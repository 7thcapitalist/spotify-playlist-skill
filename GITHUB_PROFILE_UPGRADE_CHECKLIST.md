# GitHub Profile Upgrade Checklist

This checklist is optimized to make your profile look builder-focused and product-oriented.

## 1) Curate Public Repositories

- Keep public: `spotify-playlist-skill`
- Move low-signal repos private or archive:
  - coursework/homework repos
  - asset dump repos
  - unfinished experiments with no README or clear value

## 2) Improve Profile README

Use `PROFILE_README_TEMPLATE.md` as your base in `github.com/7thcapitalist/7thcapitalist`.

Make sure it:

- has no typos
- states your focus clearly
- highlights 1-2 strong projects
- avoids beginner phrasing

## 3) Polish Flagship Repo

For `spotify-playlist-skill`:

- keep README clear and product-like
- show realistic prompt examples
- keep setup reproducible
- maintain tests and `npm run check` green

## 4) Security and Professionalism

- never commit `.env`
- keep `.env.example` with placeholders only
- rotate secrets if ever exposed
- keep commit messages clean and intentional

## 5) Add GitHub Topics (Manual in UI)

For `spotify-playlist-skill`, add topics like:

- `spotify`
- `playlist-generator`
- `typescript`
- `nodejs`
- `nlp`
- `automation`

## 6) Consistency Signals

- pin your best repos on profile
- keep one strong project better than many weak ones
- prefer quality over quantity in public code
